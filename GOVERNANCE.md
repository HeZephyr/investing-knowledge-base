# 项目治理

## 角色

- **贡献者**：通过 Issue 和 PR 提交内容、代码或审阅意见。
- **领域维护者**：负责一个市场、资产、行业或工程模块的来源质量与评审。
- **仓库维护者**：管理权限、分支保护、发布、安全事件与维护者名单。

维护者通过公开 PR 增删 `CODEOWNERS`。权限遵循最小授权；多人账户使用各自 GitHub 身份和 2FA，不共享 Token、Cookie 或 SSH 私钥。

## 决策

- 日常修复由一名有权维护者审查后合并。
- 新数据供应商、许可边界、自动交易、抓取认证、仓库公开边界和架构变更必须先有 Issue，并在 PR 中记录选择与替代方案。
- 争议优先以监管/交易所/法定披露和可复现测试裁决；仍不能消解时保留异议并标记 `disputed`。

## 发布与保护

`main` 是唯一发布分支，只能通过 PR 更新。必需状态检查、过期审批失效、对话解决和线性历史由 GitHub 分支规则强制。Pages 只部署 `main` 的已审查内容；定时任务只检查健康度，不自动合并知识结论。

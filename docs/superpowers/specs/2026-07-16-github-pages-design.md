# GitHub Pages 可视化知识库设计

## 目标

在不改变 Obsidian 使用方式的前提下，把 `wiki/`、`raw/` 与 `output/` 发布为适合新手浏览的中文静态站。站点必须免费运行、无需后端、可全文搜索，并提供知识图谱、学习进度和常用投资计算器。

## 架构

- `wiki/`、`raw/`、`output/` 是唯一内容真源。
- `investkb.site` 在构建时把内容复制到 `.site-docs/`，将 Obsidian `[[链接]]` 转为相对 Markdown 链接，并生成 `assets/data/knowledge-graph.json`。
- Material for MkDocs 负责响应式布局、中文全文搜索、目录和深浅色主题。
- 自有 JavaScript 只实现本地交互：知识图谱、复利/费用/回撤计算器、学习进度。进度只存 `localStorage`。
- GitHub Actions 构建静态产物并用官方 Pages Actions 部署；普通 CI 也执行严格站点构建，防止死链或资源缺失。

## 信息架构

- 首页：使用说明、内容统计、学习进度、风险声明与快捷入口。
- 学习中心：完整 Wiki 与推荐学习路线。
- 互动工具：复利、交易费率影响、回撤修复计算器。
- 知识图谱：节点搜索、类别过滤、拖拽/缩放、点击进入页面。
- 原始资料：官方来源、论文书籍和参考代码库卡片。
- 研究模板：公司、基金、回测、交易前检查和复盘模板。

## 边界与安全

- 不展示持仓、Cookie、Token、缓存行情或 `.env`。
- 网页不构成收益承诺或个性化投资建议。
- 不从前端直连需要认证的非公开接口。
- GitHub Pages 的访问可见性由账户套餐和仓库 Pages 设置决定；工作流本身不改变仓库的私有属性。

## 验收标准

1. 本地执行 `python -m investkb.site` 后可严格完成 `mkdocs build`。
2. 所有 Wiki 内链可在项目子路径下工作，图谱节点与链接来自实际 Wiki。
3. 三个计算器可在手机和桌面运行，异常输入有清晰提示。
4. 学习进度刷新后保留，并可一键清空。
5. 推送 `main` 自动部署 Pages，且 CI 会验证构建。

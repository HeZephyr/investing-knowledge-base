# 能源、金融与消费行业研究包设计

## 背景与授权

`config/knowledge-coverage.yaml` 显示行业维度就绪度为 18.6%，是当前最低维度。Issue #15 把本批验收边界固定为能源、金融和消费三个跨商业模式行业包。用户已明确要求持续自主推进至机器审计的 100%，并要求不要因非实质选择反复提问；因此该长期授权与 Issue 验收条件共同作为本批设计批准门槛。

本批只建设公开、可复用、教育用途的研究基础设施，不形成个股推荐，不连接券商，不保存 Cookie、Token、付费研报或第三方原始数据副本。

## 方案比较

### 方案 A：一篇跨行业长文

优点是页面少、阅读连续；缺点是来源谱系、行业差异和失败条件混在一起，无法独立审计，也不利于未来替换失效来源。

### 方案 B：先写实时 API 与行业仪表盘

优点是看起来更“量化”；缺点是当前首先缺的是指标定义、历史时点和公司映射。贸然接 API 会把免费接口波动、密钥和口径修订混进内容建设，容易制造精确但错误的结论。

### 方案 C：Raw → Wiki → Output 的三个证据包（采用）

每个行业用至少两张互补的一手 Raw 卡固定来源和限制，一张 Wiki 连接机制与财务，一张 Output 证据卡预注册可复现实证。该方案与仓库分层一致，并能用内容契约测试验证完整性。实时适配器留到 `engineering-global-data` 专门批次，避免耦合。

## 内容架构

### 能源

- Raw：EIA Open Data/API 与 Copyrights/Reuse；OPEC Annual Statistical Bulletin。
- Wiki：油气产业链、库存与备用产能、成本曲线、资本开支、裂解/炼化差异、储量与桶油经济性。
- Output：能源周期证据卡，要求冻结数据版本，区分库存变化、价格变化与公司利润，不把油价方向直接等同于股票收益。

### 金融

- Raw：BIS Basel Framework；FDIC BankFind/Call Report 数据。
- Wiki：银行为核心，并明确保险和券商是不同资产负债表；覆盖资本、流动性、信用成本、期限错配、净息差和监管口径。
- Output：银行资产负债表证据卡，要求报告期、公告可得日、重述、同业基准和压力情景。

### 消费

- Raw：美国 Census Monthly Retail Trade；中国国家统计局社会消费品零售总额方法。
- Wiki：品牌、渠道、同店/单店、销量—价格—结构、库存和营运资本；区分名义宏观销售与公司真实收入。
- Output：消费单位经济证据卡，要求同店口径、门店成熟度、促销/退货、渠道库存、汇率和并购影响。

## 统一证据规则

每张 Raw 卡必须说明：权威性、更新频率、关键字段/口径、许可或使用边界、利益冲突、历史修订以及失效条件。只保存摘要与规范链接；EIA 政府数据可按其重用政策使用并注明来源，其他来源默认 `link-and-summarize`，未知再分发权利不复制原文或数据集。

每张 Wiki 必须包含：定义与边界、产业链/商业模式、指标树、领先/同步/滞后关系、财务映射、估值、跨市场差异、反例、失败模式、检查清单、相关页面和 Raw ID。来源事实、计算和推断明确分开。

每张 Output 是空白的预注册研究合同，不伪造实证结果。它必须包含数据截止日、历史时点、数据哈希、基准、失败条件、替代解释和复现命令槽位。后续真实案例需要另一个批次用冻结公开数据完成。

## 覆盖状态设计

- `sector-energy`、`sector-financials`、`sector-consumer`：由 `missing` 提升到 `reviewed`，证据至少包含 Wiki synthesis、Raw source 与 Output template。
- `sector-framework`：本批只在三个研究包均包含实际应用骨架和独立失败复盘时提升到 `validated`。仅新增页面不算验证。
- 覆盖报告必须由 CLI 重新生成且字节级可复现。

## 错误与失效处理

- 来源 URL 失效：Raw 卡保留原 ID，新增替代版本并记录 superseded 关系，不静默覆盖。
- 数据修订：证据卡同时记录首次发布值、后续修订值和当时可得日；回测不得使用后来修订值冒充历史可得数据。
- 口径错配：宏观行业数据不能直接映射单家公司；必须验证地区、产品、渠道和收入暴露。
- 许可不明：只链接和独立摘要，不下载或再分发。
- 负结果：进入 `[[经验与失败教训]]` 或对应证据卡，不删除。

## 测试与发布

先新增内容契约测试并观察失败，再补最小内容使其通过。测试验证六张 Raw 卡的稳定 ID 与必要披露、三张 Wiki 的结构和来源、三张 Output 的预注册字段、索引可达性与覆盖状态。随后运行 Ruff、全量 Pytest、coverage validate/report、Raw/Wiki integrity、公开边界、离线研究报告和 MkDocs strict。

变更只通过 Issue #15、`codex/sector-research-packs`、PR、必需检查和 squash merge 进入 `main`。

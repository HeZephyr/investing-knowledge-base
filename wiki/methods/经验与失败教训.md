# 经验与失败教训

只记录成功会制造幸存者偏差。本页保存能够改变后续流程的错误、拒绝、负结果和修复；每条经验必须能回答：发生了什么、证据在哪里、为什么发生、影响是什么、怎样修复、怎样预防、怎样确认没有复发。

涉及账户、持仓、Cookie、Token、私人路径或未公开研究的事故只写入本地 `private/`，公开账本仅保留可复用且不泄密的模式。

## 工程经验

### 本地一键验证脚本落后于 CI 门禁

- **事件**：`scripts/verify.sh` 输出 PASS，但核对脚本发现它没有执行后来加入 CI 的 coverage、公开边界和严格站点构建。
- **证据**：`tests/test_github_workflows.py::test_local_verify_script_matches_required_ci_gates` 在修复前因缺少 `coverage validate` 失败；修复后完整脚本运行 93 项测试及全部门禁。
- **根因**：CI 工作流持续扩展，本地脚本没有能力对齐契约；原有测试只检查 CI，不检查开发者入口。
- **影响**：本地 PASS 的含义弱于 PR required checks，问题只能在远端发现，也可能造成错误的完成声明。
- **修复**：本地脚本加入 Ruff format、coverage、publication、site 生成和 MkDocs strict，并继续隔离离线报告输出。
- **预防**：用回归测试列出本地脚本必须覆盖的 CI 能力；新增 required gate 时必须同步两个入口。
- **复验**：运行 `scripts/verify.sh`，确认测试、Raw/Wiki、135 项 coverage、强密钥审计、strict site 和离线报告全部通过；不把具体测试数量写死为长期契约。

### 粗粒度清单会制造虚高完成度

- **事件**：旧清单用 42 个主题表示整个体系，单页 Wiki 加两类宽泛证据即可接近 validated，仓库就绪度达到 60.4%。
- **证据**：schema v2 将同一主题拆成内容、练习、冻结案例和持续维护四类能力，共 135 项；诚实迁移后的基线为 36.5%，其中案例验证 0/11。
- **根因**：清单把“知道概念”“会计算”“用历史可得数据做过案例”和“能持续维护”混成一个状态，分母又遗漏统计基础、公司研究和组合治理。
- **影响**：页面增长看起来像体系成熟，真实训练、复现与失败案例缺口却不可见，容易过早宣称完成。
- **修复**：引入八轴、四阶段和阶段专属证据组合；没有冻结公开报告的案例只能保持 missing。
- **预防**：新增书、论文、仓库或 Wiki 必须映射到一项原子能力；一个阶段的证据不得升级另一个阶段。
- **复验**：测试固定 135 个唯一 ID、八轴、四阶段，并验证每个 validated 项满足对应证据组合。

### 免费阅读、开放许可与机器摄取是三件事

- **事件**：OpenStax Finance 可免费在线阅读，但 2026-07-17 核验时详情页、PDF 与前言中的许可/LLM 摄取说明并不完全一致；QuantEcon 公开仓库则没有根 LICENSE。
- **证据**：`raw/books-and-papers/openstax-principles-finance.md` 与 `raw/repositories/cards/quantecon-python-intro.md` 分别记录冲突和缺失。
- **根因**：容易把“无需付费”“Creative Commons”“可复制代码”“允许自动摄取”合并成同一个许可判断。
- **影响**：直接下载、复制或让模型摄取全文可能超出作者授权；公开知识库还会把风险传播给复用者。
- **修复**：冲突或缺失时使用 `NOASSERTION`、link-only 和独立摘要；不保存 PDF、Notebook、图表、习题答案或大段正文。
- **预防**：分别核验网页、具体文件、代码、数据、品牌、教师资源和自动化条款；许可证明确前不从公开可见性推导复制权。
- **复验**：Raw 审计要求仓库卡固定 commit 与许可证状态，公开边界检查确认第三方完整仓库只在 ignored 本地审计区。

### 小练习不能替代完整能力

- **事件**：Bonferroni、简单 OLS、滚动切分和债券久期函数均通过手算测试，但它们只覆盖各主题的一部分。
- **证据**：Issue #17 的覆盖清单将假设检验、回归诊断、统计编程、债券数学和时间序列保持 `reviewed`，明确当时仍缺效应量、稳健误差、随机可重复性、凸性和模型诊断；Issue #18 只有在补齐修正久期/凸性实现与手算测试后才单独升级债券数学。
- **根因**：自动测试变绿容易诱发“整个章节已验证”的错误升级。
- **影响**：读者可能把教学原语误当生产级统计工具或完整投资研究能力。
- **修复**：按原子要求标题逐词核对证据；只在证据覆盖完整能力时 validated，部分实现保留 gap。
- **预防**：代码测试同时检查非法输入，课程页必须写常见误用；覆盖自审主动寻找标题中未实现的并列项。
- **复验**：`tests/test_foundations_curriculum.py` 同时断言已验证与 reviewed 项，不允许为了分数删除剩余缺口。

### 公开 GitHub 仓库没有许可证时只能链接

- **事件**：AI-Trader 的固定 commit 可公开读取且包含大量服务代码，但仓库根目录没有 LICENSE。
- **证据**：`raw/repositories/cards/ai-trader.md` 记录 pinned commit、`NOASSERTION` 和 link-only 决策。
- **根因**：公开可见、Star 数和可 clone 性常被误认为获得了复制、修改与再分发许可。
- **影响**：把其源码、Skill 或 API 文件打包进插件，会把不确定版权和远端认证面一并带入公开项目。
- **修复**：只保存规范链接、独立审计摘要和不可版权化的事实；不复制源码、不安装其远端 Skill。
- **预防**：仓库采用前核验根许可证、嵌套许可证、数据条款与品牌；缺失任一关键授权时默认 link-only。
- **复验**：Raw 卡与审计矩阵明确显示 `NOASSERTION`，公开边界测试确认完整 clone 不进入 Git。

### 关键词命中数不是能力覆盖率

- **事件**：离线扫描在 OpenBB 中找到 1,498 个 provider 信号路径，在 ai_quant_trade 中找到 98 个 backtest 信号路径。
- **证据**：`plugins/investing-research/scripts/audit_repository.py` 只输出分类线索；[[金融研究代码库审计]] 明确列出 false positive 与 false negative。
- **根因**：大数字容易被误读为功能数量、测试覆盖率或策略质量，文件名与文档提及却不等于实现经过验证。
- **影响**：可能在未读关键代码、未跑 fixture、未核验费用和未来函数前过度采用第三方模块。
- **修复**：扫描结果只用于定位文件；随后必须打开最小关键实现、找测试、独立手算并写采用或拒绝理由。
- **预防**：报告禁止把命中数命名为 coverage、quality 或 capability；代码事实、上游声明与推断分栏。
- **复验**：插件测试断言报告只有相对路径与计数，不含源码片段、秘密文件、绝对路径或质量结论。

### 一个单元测试目录不能代表整个平台质量

- **事件**：ai_quant_trade 材料横跨数据、因子、策略和回测，但固定 commit 中可直接识别的专门单元测试只覆盖很小一部分；AI-Trader 的服务测试也不验证获利能力。
- **证据**：四张仓库卡分别记录测试目录、fixture 和未覆盖风险，不用仓库规模或 Star 替代验收。
- **根因**：把“存在测试”从局部模块外推到异构仓库、数据源、Notebook、远端服务与交易执行层。
- **影响**：环境能安装、测试能运行或示例有收益图，仍可能隐藏数据泄漏、修订偏差、成本遗漏和不可逆订单风险。
- **修复**：按采用的最小模块建立独立契约测试；研究层与执行层硬隔离，收益声明另做冻结数据复现。
- **预防**：每次采用都写清测试的对象、输入、断言、未覆盖面和失败阈值，禁止用总测试数做背书。
- **复验**：只读插件使用自有无秘密 fixture 验证安全边界，不导入或执行第三方仓库。

### 仓库插件市场与个人插件目录不是同一个边界

- **事件**：为了让其他人能从仓库安装插件，市场清单必须随项目版本化；个人安装状态则属于本机 Codex 配置。
- **证据**：`.agents/plugins/marketplace.json` 指向版本化的 `plugins/investing-research/`，安装文档把 marketplace add 与 plugin add 分成两步。
- **根因**：容易把“插件源码放仓库”“市场能发现插件”和“当前电脑已启用插件”混成同一状态。
- **影响**：只提交源码会让用户找不到安装入口；把个人缓存或认证提交进仓库又会泄漏本机状态。
- **修复**：仓库只提交清单、插件、Skill、脚本与非敏感 fixture；个人安装记录留在 Codex 管理目录。
- **预防**：发布同时运行官方插件 validator、Skill validator、离线测试和真实 CLI 列表检查。
- **复验**：全新本地安装能通过仓库 marketplace 发现插件，Git 状态仍不出现个人配置、Cookie 或 Token。

### 精确 Action 版本断言使正常升级失败

- **事件**：Dependabot 更新 GitHub Pages Actions 主版本后，Python 测试失败，而站点构建、Raw/Wiki、回测和公开边界检查正常。
- **证据**：PR #5 的失败日志指向 `tests/test_site.py` 对 `configure-pages@v5` 的精确字符串断言；PR #8 修复后新版 Actions 全绿。
- **根因**：测试绑定了依赖实现版本，没有验证真正的能力与最低安全契约。
- **影响**：合法安全升级被阻塞，形成“自动更新必红”的维护债务。
- **修复**：验证官方 Action 名称、`@vN` 格式和最低主版本，同时接受更高合法版本。
- **预防**：依赖测试优先验证行为、来源和安全下限，不把动态版本或构建哈希写死。
- **复验**：让 Dependabot 分支 rebase 后重新运行全部 required checks，并成功合并。

### 公开可访问不等于允许深链或再分发

- **事件**：GMO Research Library 对研究有价值，但合并前自审发现网站条款限制未经同意的公共深链和复制。
- **证据**：`raw/experts/cards/gmo-research-library.md` 的许可段及 `tests/test_raw_collection.py` 的根入口回归约束。
- **根因**：初始设计把“无需登录”误当作“适合公开知识库链接与分析”。
- **影响**：若机械摄取，可能让公开仓库违反来源条款。
- **修复**：降级为根入口元数据，不深链研究库、不复制报告或专有模型。
- **预防**：摄取前同时检查访问、版权、数据许可、深链、自动化和再分发条款；条款不清时选择更保守用途。
- **复验**：来源测试要求 GMO 卡 URL 只能是网站根入口，且许可段必须明确“不深链”。

### SSE 心跳不是行情数据

- **事件**：网页 Network 中出现 `next_seq` 和 `noop` 消息，连接状态为 200，但 `content` 为空。
- **证据**：消息语义只推进序列或维持长连接，没有价格、成交量或时间序列字段。
- **根因**：把传输层活动误判成业务数据载荷。
- **影响**：若直接作为行情接口，会产生空数据、重复轮询或虚假的“实时成功”。
- **修复**：先定义业务字段契约；心跳、游标、订阅确认与行情 payload 分开解析。
- **预防**：数据适配器必须对空响应、字段漂移和非业务事件显式失败，不以 HTTP 200 证明数据有效。
- **复验**：只有包含标准化日期、价格、成交量且通过 [[数据质量]] 校验的响应才能进入缓存。

### 公开网页返回 403 不是绕过认证的邀请

- **事件**：OPEC 与 Census 的资料可通过公开浏览入口读取，但 2026-07-17 的通用命令行请求返回 HTTP 403。
- **证据**：相应 Raw 卡的“局限与失效条件”记录访问方式差异；其他四个行业官方入口同轮命令行检查返回 200。
- **根因**：网页/CDN 的自动化访问策略与内容公开性是两个维度，不能从浏览器可见推断任意客户端都被允许。
- **影响**：把 403 当作数据不存在会丢失来源；用 Cookie 或模拟会话强行绕过则增加条款、凭据泄漏和维护风险。
- **修复**：OPEC 降为 link-only 摘要；Census 自动化改走官方 Economic Indicators API 或公开下载。
- **预防**：先找官方 API/批量下载与许可说明；没有合规机器入口时保持人工链接，不把会话材料写入代码。
- **复验**：定时检查分别报告“浏览入口”“官方 API”和“自动下载”能力，不用单一 HTTP 状态代表全部可用性。

## 研究经验

### 未来生效规则不能提前写成当前制度

- **事件**：欧盟已达成从 T+2 转向 T+1 的政治协议并给出 2027-10-11 目标日，但 2026-07-17 的当前一般结算仍按 T+2 理解且未来范围存在例外。
- **证据**：`raw/official/europe/eu-transparency-settlement.md` 与 [[欧盟市场]] 分开记录现行、协议和目标生效状态。
- **根因**：新闻标题、立法提案、政治协议、Official Journal 公布和实际适用是不同阶段，容易被压缩成一个“最新规则”。
- **影响**：提前用 T+1 回测会错配现金、外汇、公司行动和交收失败风险，也会把未来信息带入历史。
- **修复**：规则事件同时保存提案日、协议日、公布日、生效日、适用日与例外；模拟只使用交易日当时已适用状态。
- **预防**：所有动态制度卡必须有核验日和失效条件，未来规则用“计划/目标”而非现在时。
- **复验**：站点与测试保留 T+2/T+1 双状态，规则转换后再以官方正式文本更新而不改写旧记录。

### 市场分类不等于法律和操作可达

- **事件**：IMF、World Bank、BIS 和指数提供者对“新兴市场”的范围与目的不同，且 GFDD 最新公开版本明显滞后于当前交易状态。
- **证据**：`raw/official/global/imf-areaer.md`、`world-bank-gfdd.md`、`bis-debt-statistics.md` 和 [[新兴市场访问]] 将标签、资本限制、市场结构与证券交易链分开。
- **根因**：国家标签便于讲故事，但隐藏外资账户、持股上限、在岸/离岸汇率、托管、汇回和临时资本措施。
- **影响**：指数中存在某国证券不代表用户可按指数价格买卖或退出；用当前分类回测历史还会产生前视偏差。
- **修复**：每个国家建立法律准入、操作准入、可兑换、容量和压力退出五层访问卡；分类保存公告与生效日。
- **预防**：跨国数据必须带版本、指标代码、截止日、底层来源和缺失规则，不能将年度结构数据伪装成实时信号。
- **复验**：只有当地监管、交易所、CSD、税务和券商可用性均核验后，才能从“研究覆盖”升级为“可交易”。

### 连续期货曲线不是可以直接成交的资产

- **事件**：商品研究常用回溯调整连续合约画出平滑历史，但真实头寸持有具体到期月并在通知/交割前换月。
- **证据**：[[大宗商品]] 和 `raw/official/united-states/cftc-futures-basics.md` 要求保存原始合约链、乘数、换月规则、保证金与通知日。
- **根因**：连续价格方便统计，却会隐藏跳空调整、展期价差、流动性迁移和不可交割约束。
- **影响**：直接对连续价格算收益可制造不存在的盈亏，忽略保证金现金流并可能持仓进入交割。
- **修复**：用逐月原始合约重建持仓和换月成交；价差调整只用于图形/特征时明确标记，不用于现金盈亏。
- **预防**：商品报告强制披露合约代码、最后交易/通知日、换月触发、成本、曲线和数据许可。
- **复验**：手算基差/展期练习与原始合约链回放结果一致后，才允许进入冻结案例。

### 更新后的历史序列会制造时间偏差

- **事件**：AQR Data Library 同时提供冻结论文样本和持续更新扩展；更新扩展可能因供应商修订重写整段历史。
- **证据**：`raw/experts/cards/aqr-data-library.md` 记录了冻结/扩展区别、月度更新和时间偏差。
- **根因**：研究者容易把今天下载的干净历史当成过去当时已经可得的数据。
- **影响**：回测可能使用未来修订，夸大稳定性和可复现性。
- **修复**：保存抓取时间、文件哈希、样本截止日和版本；优先用冻结论文样本做复现。
- **预防**：所有因子研究填写 [[投资研究证据矩阵]] 和实证证据卡的历史时点字段。
- **复验**：用独立快照重跑，比较修订前后结果并报告差异。

### 负结果和数据事故也必须入库

- **事件**：只保留成功策略、正收益参数或最终清洗数据，会让知识库看起来比真实研究可靠。
- **证据**：`output/templates/实证证据卡.md` 已要求填写预设失效条件、失败结果、数据事故和尝试次数。
- **根因**：结果导向、发表偏差和事后挑选让失败过程被删除。
- **影响**：团队重复踩坑，样本外表现恶化，无法估计真实尝试次数。
- **修复**：失败命题保留原始预注册、错误类型、修复与是否放弃；不因结果不好改写成功阈值。
- **预防**：正式研究先预注册，Output 报告同时发布基准、成本、敏感性和负结果摘要。
- **复验**：后续独立样本只能确认、缩小或推翻命题，不能覆盖旧结论。

### 宏观行业增长不能直接替代公司收入

- **事件**：能源库存、社会零售或美国零售销售能描述聚合行业状态，但缺少地区、产品、渠道、价格和公司份额映射。
- **证据**：`wiki/sectors/能源.md` 与 `wiki/sectors/消费.md` 均要求把宏观量连接到公司实现价、出货、渠道库存和法定披露。
- **根因**：聚合序列容易获得且叙事直观，研究者会跳过法律实体与收入暴露核验。
- **影响**：宏观方向判断正确，个股收入、利润甚至股价结论仍可能完全错误。
- **修复**：强制建立“宏观 → 行业环节 → 公司暴露 → 三表 → 估值”桥，并单列份额、汇率、套保、渠道库存和会计日历。
- **预防**：行业证据卡必须填写公司映射权重和“宏观正确、公司错误”的反例。
- **复验**：只有公司公告数据能与聚合指标按历史可得日勾稽时，才允许进入公司结论。

### Advance 与最终修订序列会制造历史时点泄漏

- **事件**：Census 的 advance 零售估计随后由更大样本和年度基准修订；能源与国家统计序列也可能更正口径或历史值。
- **证据**：六张行业 Raw 卡都设置了“历史修订”段，三张行业证据卡要求同时记录数据截止日、历史可得日、版本与 SHA-256。
- **根因**：今天下载的最终序列更整洁，常被误认为过去决策时已经存在。
- **影响**：回测获得不可交易的信息优势，参数稳定性和预测力被夸大。
- **修复**：保存 release vintage、首次发布时间、抓取时间和内容哈希；无法获得历史 vintage 时明确降级为解释性研究。
- **预防**：测试与报告分开 current/final 与 as-released 数据，不允许静默替换。
- **复验**：用首次发布值与最终值分别重跑，报告结论方向、显著性和交易结果差异。

### 权威来源仍有目标、范围与激励边界

- **事件**：OPEC 是生产者组织，BIS/FDIC 是监管者，宏观统计机构为总体测量服务；它们都不是股票研究机构。
- **证据**：相应 Raw 卡分别记录生产者利益、监管目标、机构覆盖和抽样口径。
- **根因**：把 A 级来源误解为“任何投资问题上的无条件真相”，忽略发布者目标和测量对象。
- **影响**：单一来源的估算、分类或政策口径被过度外推到公司价值。
- **修复**：保留来源等级，同时强制写利益冲突、覆盖范围、替代来源和失效条件。
- **预防**：关键结论至少用另一类来源或公司法定披露交叉验证。
- **复验**：若不同来源冲突，保留 disputed 状态，不能按叙事偏好选择数据。

### 同名 Basel 指标不保证跨司法辖区可比

- **事件**：银行都披露 CET1 或 RWA，但本地实施版本、监管扣减、模型、过渡安排和合并范围可能不同。
- **证据**：`raw/official/global/bis-basel-framework.md` 与 `wiki/sectors/金融.md` 要求记录章节生效日和当地实施文本。
- **根因**：缩写相同造成虚假标准化，研究者省略制度版本与法律实体映射。
- **影响**：跨国资本率排名、压力测试和 P/B 比较出现系统性口径错误。
- **修复**：统一报告期、实体、币种、分子、分母和过渡安排，再做比较；无法统一则并列展示。
- **预防**：银行证据卡把本地实施版本和 Basel 差异设为必填项。
- **复验**：用监管原表重算至少一个资本指标，并与公司披露差异表勾稽。

## 使用规则

### README 的许可证句子不能替代许可证证据

- **事件**：finance-quant-skills README 页尾声称 MIT，但固定 commit 没有根 LICENSE/COPYING/NOTICE，包元数据也未声明 license；仓库还含可能来自官方文档和其他 Skill 的材料。
- **证据**：`raw/repositories/cards/finance-quant-skills.md` 与 [[金融量化平台与Skill审计|金融量化平台与 Skill 审计]] 分开记录仓库声明、文件事实和复制内容谱系。
- **根因**：把“公开可读”“README 写 MIT”和“所有嵌套内容均可 MIT 再发布”错误地合并。
- **影响**：整体安装或再打包可能引入无法证明的版权与供应链义务。
- **修复**：根许可缺失时记 `NOASSERTION`；代码、文档、图片、模板、数据和 imported Skill 分别核验。
- **预防**：审计测试固定许可证结论，任何升级必须引用新许可证文件与 commit。
- **复验**：上游补齐许可证后，仍逐资源复核来源，再决定是否从 link-only 晋级。

### 构建和 Skill 文本都是执行面

- **事件**：FinHack `setup.py` 在构建/安装前会创建文件并覆盖包版本文件；finance-quant-skills 的 Markdown 指令可要求安装依赖、复制 Cookie、连接终端和下单。
- **证据**：`raw/repositories/cards/finhack.md`、`finance-quant-skills.md` 和插件 `$audit-finance-skills` 的硬安全边界。
- **根因**：把 packaging 当静态元数据，把 Markdown 当无害文档，忽略代理会执行自然语言指令。
- **影响**：仅为“看看能否安装”就可能修改工作树、运行不可信代码、泄漏会话或触发资金操作。
- **修复**：默认离线静态审计；不执行 setup、hook、script 或上游 Skill；Cookie 与订单能力 quarantine。
- **预防**：所有第三方 Skill 审计强制检查 install-time effects、prompt injection、credential 和 money-moving surface。
- **复验**：官方 Skill validator、离线测试和公开边界检查必须同时通过；第三方代码不进入 Git。

### 本地 marketplace 不能使用 Git upgrade 流程

- **事件**：为插件 v0.2 更新 `personal` marketplace 时，`codex plugin marketplace upgrade personal` 明确报错：该 marketplace 不是 Git marketplace。
- **证据**：`codex plugin marketplace list` 显示 `personal` 根为本地仓库路径；CLI help 说明 upgrade 只刷新 Git snapshot。
- **根因**：旧 README 把本地路径 marketplace 与 Git marketplace 的更新命令合并成一个流程。
- **影响**：插件仍显示旧 cache 版本，用户误以为新 Skill 已加载。
- **修复**：本地 marketplace 使用 `plugin remove` 后 `plugin add`；Git marketplace 才先 `marketplace upgrade`。
- **预防**：更新文档按 source type 分支，并以 `codex plugin list` 的版本为验收证据。
- **复验**：重装后版本必须等于 manifest 的 `0.2.0+codex.*`；新 task 再加载 Skill。

### 当前修订历史不等于历史首次发布值

- **事件**：能源和消费案例通过 FRED 下载到当前修订的年度/月度历史，但源接口没有同时返回每个观测的首次发布版本。
- **证据**：`raw/cases/energy/manifest.yaml` 与 `raw/cases/consumer/manifest.yaml` 将 `available_at` 明确标为本次冻结快照的可得日，报告也声明不是 vintage/realtime-start 证据。
- **根因**：把“决策日之前能够看到”误当成“历史当时就以同一个值发布”，忽略宏观序列会修订。
- **影响**：若把最终修订值回填到历史回测，会形成不易察觉的未来信息并夸大策略表现。
- **修复**：本轮只做 2026-07-17 截面的描述性案例；需要历史模拟时改用 ALFRED/vintage 或官方首次发布档案。
- **预防**：冻结清单同时记录决策日、首次可得口径、抓取时间和哈希；报告必须写明 revision 限制。
- **复验**：离线验证拒绝晚于决策日的观测；后续 vintage 案例需测试 realtime-start 与发布日期。

### 公开端点受阻不授权 Cookie 绕过

- **事件**：SEC companyfacts 和 Census 历史文件在本环境返回 403/连接关闭，浏览器也无法打开 Census 目录。
- **证据**：案例最终使用无需登录的 FRED/FDIC 接口与公司法定披露页，没有提交 Cookie、Token、完整响应或浏览器会话。
- **根因**：公开网页仍可能有流量控制、网络边缘策略或自动化访问限制；“公开”不等于允许规避访问控制。
- **影响**：若为赶进度复制个人 Cookie，既可能泄漏账户，也会让公开仓库无法复现并违反来源条款。
- **修复**：切换到等价的公开官方来源，缩小为合法的规范化快照，并保留来源失败与替代路径。
- **预防**：认证材料一律进私有隔离层且默认不用；公开案例必须有无账号的复现路径。
- **复验**：公开边界扫描、离线测试和 Git 历史均不得出现会话值；无网络环境仍能重算结果。

### Arrow 字符串列不能原地写入日期对象

- **事件**：临时数据整理脚本把 pandas Arrow 字符串列原地赋为 datetime 时触发类型错误。
- **证据**：失败发生在生成年度汇总的中间分析，修复后以新日期列解析，最终提交的是稳定 CSV 与独立校验器。
- **根因**：Arrow-backed dtype 比 object dtype 更严格，原地赋值不会自动把整列安全提升为 datetime。
- **影响**：一次性脚本中断；若静默强制转换，还可能产生缺失日期或错误年度分组。
- **修复**：使用 `assign`/新列承接 `to_datetime` 结果，再显式提取年份并检查缺失值。
- **预防**：分析脚本不依赖隐式 dtype 变化；转换后断言类型、空值和行数。
- **复验**：冻结快照测试覆盖行数、重复键、数值有限性、UTC 时间和决策日边界。

### 监管批准、支付覆盖与需求是三道门槛

- **事件**：Aduhelm 获 FDA accelerated approval，但 CMS 随后把覆盖限制在合资格试验，公司记录与需求预期相关的库存/采购承诺减值。
- **证据**：`raw/cases/healthcare/manifest.yaml` 将 FDA、CMS 和 Biogen 三个事件分开冻结，`output/cases/healthcare.md` 按预设失败条件计算。
- **根因**：医药叙事把监管的安全有效判断、payer 的 reasonable-and-necessary 判断和真实采用错误地合并。
- **影响**：峰值销售、库存、产能和 rNPV 被系统性高估，surrogate 与临床获益不确定性被隐藏。
- **修复**：估值树增加 approval、label、coverage、diagnosis/capacity、initiation 和 persistence 独立分支。
- **预防**：医药预注册卡要求分别记录监管、支付、患者与公司财务结果。
- **复验**：离线案例必须同时返回 approval、coverage 和 write-off，且完整命题为负。

### 运营指标的不同时间窗不能硬拼成因果桥

- **事件**：Meta DAP 是 12 月平均，而收入、广告展示和每广告价格是全年同比；它们可做诊断拆解但不是完全对齐的因果实验。
- **证据**：`raw/cases/internet/manifest.yaml` 保留每个指标的期间和发布日期，报告明确只做近似桥。
- **根因**：图表把“同一年度标签”误当成同一测量窗口，并忽略用户定义、地域和产品组合。
- **影响**：用户增长被错误归因成全部收入增长，网络效应和变现能力被夸大。
- **修复**：比较前固定分母、窗口、去重和单位；无法对齐时保留方向/幅度诊断但不声称精确贡献。
- **预防**：平台证据卡把用户、展示、价格、收入和口径变更设为独立字段。
- **复验**：测试重算 5.02% DAP、21.94% 收入及 11%/10% 广告分量，并判定单变量命题失败。

### 库存金额不是单一周期时钟

- **事件**：Micron FY2024 收入和毛利大幅修复时，期末库存金额仍比 FY2023 高 5.82%。
- **证据**：`raw/cases/memory/observations.csv` 同时保存收入、毛利、库存、NRV 减值、Capex 与 ASP/bit 方向。
- **根因**：库存金额混合数量、成本、产品组合、在制品和减值，且原厂、渠道、客户库存处于不同位置。
- **影响**：只等库存绝对下降会错过价格/组合修复；只看利润又会忽略减值基数和现金占用。
- **修复**：库存金额与天数/结构/减值、ASP、bit shipment、渠道/客户库存和现金流并列。
- **预防**：存储证据卡禁止用一个现货价或库存金额代表完整 DRAM/NAND/HBM 周期。
- **复验**：测试确认收入 +61.59%、毛利 +31.5pp、库存 +5.82%，并保留负结果。

### 算术平均、全期复合收益与 CAGR 不能混名

- **事件**：HML 复现的第一版实现把全期复合收益写成了年化几何收益，虽然公式可算，却与预注册指标不是同一个量。
- **证据**：2000—2025 年 HML 年度算术均值为 1.95%，全期复合收益为 7.97%；两者均由冻结年度序列重算。
- **根因**：`compound return` 的口语歧义掩盖了累计乘积与其年化根之间的区别。
- **影响**：指标标签与数值错配会让读者错误比较收益水平，并可能改变成功门槛。
- **修复**：代码与报告分别命名 arithmetic mean、cumulative compound return 和 CAGR；测试锁定定义与数值。
- **预防**：预注册必须同时写公式、期间和单位，不只写指标简称。
- **复验**：离线测试独立检查 26 年、14 个正年份、1.95% 与 7.97%。

### 当前因子 vintage 可复现但不等于永恒历史

- **事件**：Kenneth French 官方文件明确使用 2026-05 CRSP database，历史因子以后仍可能修订。
- **证据**：manifest 固定 vintage、官方 ZIP 与最小年度派生事实，快照以 SHA-256 锁定。
- **根因**：研究数据常被误当成不会变化的静态真相，且可下载不等于可以完整再分发。
- **影响**：不同日期复现可能得到不同尾数；复制完整档案还会扩大版权与维护风险。
- **修复**：只提交支持案例的最小事实，链接官方原档并声明版权和版本。
- **预防**：所有研究数据写 provider、vintage、抓取日、期间、哈希和再分发边界。
- **复验**：离线结果必须从同一哈希重现；新 vintage 只能作为显式更新 PR。

### 合法公开备选优于个人 Cookie

- **事件**：Eastmoney ETF 历史接口在当前环境发生代理错误，而 AKShare 的无认证 Sina 接口可正常取得公开价格。
- **证据**：portfolio manifest 记录接口、代码、期间、抓取日与失败后的替代来源，仓库没有 Cookie 或 Token。
- **根因**：把某一 provider 失败误判成“必须借用登录会话”，忽略等价公开来源。
- **影响**：Cookie 会泄漏账户并破坏他人复现；后台接口也可能违反使用边界或随时变化。
- **修复**：按 provider fallback 顺序切换到无需认证的文档化接口，只冻结最小数据。
- **预防**：公开案例默认禁止个人会话材料；来源失败进入日志，不扩大访问权限。
- **复验**：公开边界扫描、Git 历史和离线测试同时通过。

### 未复权 ETF 价格不是总收益

- **事件**：Sina 年末收盘价可以验证再平衡算法，但无法证明分红、利息分配和复权口径。
- **证据**：三资产组合报告将结果标成 price-return proxy，并同时公开六年小样本与端点敏感性。
- **根因**：价格序列容易被直接当成持有者总回报，尤其会低估股票和债券基金的分配收益。
- **影响**：资产和基准的收益比较可能偏差；61.66% 正结果不能转成未来赚钱承诺。
- **修复**：案例只验证权重漂移、换手费用、波动和回撤计算，不声称可投资总收益。
- **预防**：正式组合报告必须证明 adjustment/distribution 口径，否则标题和指标明确标为价格代理。
- **复验**：测试锁定价格代理结果；获得权威总收益数据后另建版本，不覆盖旧快照。

### 报告期不是信息可得日

- **事件**：同一 2023 收入可能在 2024 年初首次提交、八个月后重述；若历史模型永远读取最新值，就会回填未来信息。
- **证据**：`FilingFact` 同时保存 period、filed_at 和 accession，测试在重述前后分别返回原值和修订值。
- **根因**：数据表只用 `company + fiscal_period` 作主键，覆盖了版本与提交时点。
- **影响**：历史筛选、回测和研究日志看似准确，实则使用当时未知的重述。
- **修复**：主键扩展为指标、期间、提交时间和版本；决策时间必须带时区。
- **预防**：抓取法定披露时保存 form/公告类别、accession、filed_at、amendment 与原始链接。
- **复验**：未来提交被排除，无时区决策时间被拒绝，旧版本不删除。

### 三表差额不能塞进“其他”

- **事件**：宽松模型常用一个平衡项强行令资产负债表或现金桥相等，隐藏单位、符号、范围和漏项错误。
- **证据**：`reconcile_statements` 对资产恒等式和现金流恒等式分别计算 gap，超过容差立即失败。
- **根因**：把勾稽当展示格式，而不是数据质量合同。
- **影响**：后续 FCF、净债务和估值建立在不可解释的差额上。
- **修复**：fail closed，并要求研究者回到现金等价物、汇率、并购、非现金项目和重分类附注。
- **预防**：禁止无来源 plug；每个调整保留名称、方向、单位和原始脚注。
- **复验**：正确案例两个 gap 都为 0，期末现金相差 ±1 都必须报错。

### 负利润倍数应保持未定义

- **事件**：亏损公司若仍输出负 P/E 或负 EV/EBITDA，排序器会把数学结果误当成便宜程度。
- **证据**：`normalized_multiples` 对非正 EBITDA 和净利润返回 `None`，但保留正收入的 EV/Revenue。
- **根因**：公式执行没有先检查分母经济含义。
- **影响**：负倍数在筛选和图表中产生虚假可比性。
- **修复**：非正分母显式 undefined，改用单位经济、毛利、现金消耗或达到盈亏平衡所需假设。
- **预防**：每个倍数定义可用范围、会计调整和周期归一化规则。
- **复验**：亏损 fixture 只返回 2.0x EV/Revenue，另外两个字段为空。

### 反向 DCF 必须有可解区间

- **事件**：若目标企业价值超过增长上下界能产生的价值，数值求解器可能仍返回一个看似精确的边界答案。
- **证据**：`reverse_dcf_growth` 先计算上下界价值，目标未被 bracket 时明确失败。
- **根因**：把求根算法收敛等同于经济假设合理，忽略 WACC、终值增长与 FCF margin 约束。
- **影响**：极端价格被翻译成伪精确增长率，掩盖模型结构不适用。
- **修复**：要求 WACC > terminal growth、正收入/FCF margin 和显式增长区间，再二分求解。
- **预防**：报告同时展示输入、区间、终值占比和不可解状态，不只展示一个百分比。
- **复验**：已知 8% fixture 可恢复，超出区间的 100000 企业价值必须报 `not bracketed`。

### 外部现金流不能让交易成本少算一半

- **事件**：再平衡第一版同时使用“单边换手”口径和费用口径，若直接对绝对交易额除二后收费，会漏掉现金流投入产生的实际买入。
- **证据**：100 资产加 10 现金后，目标交易为 -5/+3/+12，总成交额 20、单边换手额 10；20bp 费用应为 0.04。
- **根因**：把自融资组合常用的 half-turnover 当成券商实际收费本金。
- **影响**：策略频繁投入/提款时系统性低估成本，回测净值偏高。
- **修复**：报告 `gross_traded_value` 与 one-way turnover 两个字段；费用按 gross buy+sell 计算。
- **预防**：每个成本公式明确收费方向、外部现金流、税与最小费用。
- **复验**：测试锁定 20、10/110、0.04 和 109.96 四个结果。

### 协方差矩阵不是任意数字表

- **事件**：风险贡献只有在协方差有限、对称且半正定时才有一致的组合方差解释。
- **证据**：`risk_contributions` 检查形状、对称性和最小特征值，成分风险必须加总到组合波动。
- **根因**：缺失值填充、不同窗口拼接或人工修改会破坏矩阵结构。
- **影响**：得到负方差、虚假分散或不稳定的风险预算。
- **修复**：非法矩阵 fail closed；零波动时不输出伪造的百分比贡献。
- **预防**：固定估计频率、窗口、币种和缺失值规则，并保留收缩/修正方法。
- **复验**：合法 2×2 fixture 加总一致，非对称矩阵被拒绝。

### 归因必须加回主动收益

- **事件**：配置、选择和交互效应若公式版本或权重口径混用，会与组合减基准的主动收益不相等。
- **证据**：单期 Brinson-Fachler fixture 的三项效应严格加总 1.5%。
- **根因**：把 Brinson-Hood-Beebower 与 Brinson-Fachler 配置项混写，或漏掉交互项。
- **影响**：漂亮的归因图无法解释真实超额，管理人评价失真。
- **修复**：实现固定公式版本并设置 reconciliation tolerance；多期另选链接方法。
- **预防**：报告写清基准、总收益、币种、费用、现金和公式版本。
- **复验**：任何残差超过容差都失败，不塞入“其他”。

### 决策日志只评分过程完整度

- **事件**：若复盘分数依赖最终盈亏，会奖励幸运的坏决策并惩罚合理但遭遇尾部事件的决策。
- **证据**：公开 journal validator 只检查日期、命题、基准率、成功/失败条件、期限、上限和证据，不读取收益。
- **根因**：结果偏差把 outcome 当成 decision quality。
- **影响**：研究者事后提高信心、延长期限或删除负结果，无法学习。
- **修复**：结果出现前冻结字段，结果后分别评价过程与校准。
- **预防**：私人日志留在忽略层；公开测试只用 paper portfolio，无持仓。
- **复验**：完整记录得 1.0，只有日期的记录明确列出缺失字段。

1. 新事故复制 `output/templates/研究事故与教训.md`；私人事故放入忽略的本地层。
2. 修复必须关联测试、CI run、数据哈希或明确的人工复验步骤。
3. 同类事件再次发生时追加新记录并链接旧条目，不把旧失败改写成“从未发生”。
4. 每次大版本发布回顾本页，把仍有效的预防规则写入 `AGENTS.md`、测试或覆盖清单。

# 私人研究工作区

个人目标、观察池、持仓成本和决策记录属于本地 `private/`，永不进入 Git、PR、Pages、公开报告或模型上下文摘要。用户目前没有持仓是合法状态：空持仓不等于缺功能，也不应该用虚构证券填满模板。

## 初始化与验证

```bash
investkb private init --root .
investkb private validate --root .
```

初始化器只创建 `policy.yaml`、`watchlist.yaml`、`positions.yaml` 和 `decision-journal.yaml`，已有目录时拒绝覆盖。校验命令只输出观察项、持仓和日志条数，不回显内容。测试全部在临时目录运行，所以公共仓库本身不携带任何个人示例。

## 四份文档的职责

| 文档 | 保存什么 | 不保存什么 |
|---|---|---|
| policy | 本位币、期限、可承受回撤、允许市场、禁止行为 | 密码、券商账号、API Key |
| watchlist | 证券身份、原命题、反证、研究状态 | “必涨”标签、自动订单 |
| positions | 数量、平均成本、币种、取得时间 | Cookie、Token、broker session |
| decision-journal | 决策时点、证据、反证和事后结果 | 删除失败记录后的美化历史 |

所有 YAML 都使用 allow-list schema 和 `safe_load`。必填字段、非空证券身份、状态枚举、ISO 日期、正持仓数量、非负成本、三位大写币种和投资政策数值边界逐项验证；目录/文件 symlink、额外文件、未知字段以及名称含 Cookie、Token、secret、credential、broker 或订单 endpoint 的字段会失败。自动交易、杠杆和做空是不可删除的最低禁止集。

发布审计通过 Git index 区分两种情况：被 `.gitignore` 排除的本地私人文件完全跳过且不读取；即使用 `git add -f` 强行跟踪的私人路径仍会阻断发布。这样初始化后全量测试可继续运行，同时不能用 ignore 规则掩盖已跟踪泄漏。

## 正确工作流

1. 先在 policy 写目标、期限和损失边界，仍不需要建立持仓。
2. 观察池先写可证伪原命题和反证；公开事实回链到 Raw/Wiki。
3. 真正成交后才写 positions，证券身份、币种和成本口径必须明确。
4. 决策日志保留“没买、放弃、判断错误”和数据事故，按过程而非短期盈亏复盘。
5. 提交前仍运行公开边界审计；任何需要分享的经验先去身份化并重新写入 [[经验与失败教训]]。

## 失败模式

- 把私人文件复制到 `output/` 以便网页查看，导致 Pages 泄漏。
- 用 Cookie/Token 方便抓取，随后把 session 材料混入配置或日志。
- 没有持仓时虚构样本，后来忘记它是演示数据。
- 只记录买入理由，不写反证和放弃条件；事后按涨跌改写原命题。
- 把研究助手直接连到 broker/order endpoint，使阅读材料变成资金权限。

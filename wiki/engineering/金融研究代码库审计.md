# 金融研究代码库审计

本页回答的不是“哪个项目 Star 多”，而是每个项目能为证据化投资研究复用什么、需要什么认证、哪里可能失效、哪些能力不能进入默认插件。审计固定到 2026-07-17 的 commit；上游更新后必须重新运行，不把旧结论套到新代码。

## 一页结论

| 项目 | 代码角色 | 根许可证 | 强项 | 最大边界 | 决策 |
|---|---|---|---|---|---|
| OpenBB | 多 provider 的标准化数据层 | AGPL-3.0-only | 标准模型、provider/extension、Python/REST 输出 | 数据条款与 Key 独立；AGPL 网络服务义务 | 采用架构，可选只读依赖 |
| myhhub/stock | A 股抓取、指标、筛选、回测、Web/交易一体化 | Apache-2.0 | A 股领域目录与策略/形态覆盖 | 网页接口稳定性、公式声明、自动交易风险 | 重写最小算法，不启交易 |
| AI-Trader | 远端 agent/live/copy trading 平台 | NOASSERTION | agent 事件、服务测试与实验元数据线索 | 无根许可证；注册、Token、broker sync、copy trade | link-only，隔离 |
| ai_quant_trade | 大型量化知识与示例集合 | Apache-2.0 | 数据—因子—策略—回测的广泛样例 | 环境异构、第三方许可、收益/付费宣传、测试不均 | 逐示例审计 |

代码许可证只回答代码怎么用，不回答行情、财报、新闻、图片、论文、模型权重和外部服务能否再分发。公开 GitHub 也不等于允许复制；AI-Trader 没有根许可证时，不安装其 Skill 或 API 文件。

## OpenBB 真正解决什么

OpenBB 的可复用核心不是“装一个包就拥有 Bloomberg 全部数据”，而是统一接口：标准 `QueryParams` 描述输入，标准 `Data` 描述输出，provider 实现供应商差异，extension/router 暴露主题命令，结果对象再转 DataFrame、API 或 UI。这个分层适合知识库的免费数据适配层。

固定 commit 的 README 把 provider 最低订阅层分成：

- **None**：例如 IMF、EconDB、SEC、yfinance 等入口，上游表中没有最低订阅要求；仍可能有网站条款、限频或非官方数据风险。
- **Free/None-Free**：例如 BLS、Congress、CFTC、FMP、FRED、OECD、Polygon、Tiingo、FINRA、Nasdaq 等；“Free”可能仍需注册/API Key，免费额度与字段会变。
- **Paid**：例如 Benzinga、Intrinio、TradingEconomics 等；OpenBB 不替用户购买或授予数据权。

因此安装 `openbb` 只安装代码。每次查询仍要记录 provider、endpoint、参数、时间、时区、复权、公司行动、历史修订、行数和内容哈希。对于 A 股/港股，不把 OpenBB 当唯一覆盖层；现有 AKShare/BaoStock 与官方披露路径继续独立保留。

## 对“200+ 选股条件”的可证伪检查

myhhub/stock 的维度和目录很有发现价值，但声明必须拆成测试：

1. 每个字段的来源、日期、单位、缺失与可得时间是什么？
2. MACD/KDJ/BOLL/RSI 等公式使用哪个初始化、移动平均、复权和舍入？
3. 所谓“与同花顺/通达信一致”比较了哪些代码、日期和版本，容差是多少？
4. 筹码分布是交易所事实还是模型估计？自由流通盘、分红送转和停牌怎样处理？
5. 回测是否包含退市、涨跌停、不可成交、次日成交、费用、滑点和历史成分？
6. 在 200 多维度与多策略中尝试了多少组合，如何控制数据窥探？

筛选器只能缩小研究对象，不能产生买入结论。任何形态命中都要在未参与选择的时间后样本、费用后基准和负结果集合中验证。

## AI 与自动交易声明怎么审

“AI”“fully automated”“高频”“年化 53%”“复制高手”不是证据等级。AI-Trader 的代码显示认证、远端注册、copytrade/tradesync 与实时价格防护；这证明它有资金操作面，不证明 agent 能稳定获利。ai_quant_trade 的目录展示很多算法和示例，但统一安装依赖、运行一个 Notebook 或复现一张收益图都不能证明平台级质量。

每个 AI 交易命题至少回答：

- 训练/验证/测试按时间如何切分，有无全样本归一化或标签泄漏？
- 模型比较了什么简单基线，超参数和提示词尝试次数是多少？
- 奖励是否包含费用、滑点、容量、拒单、停牌和尾部损失？
- 数据是否历史时点可得，退市与公司行动是否存在？
- agent 能调用哪些工具，是否存在提示注入、越权、Token 泄漏和不可逆订单？
- 结果能否用固定 seed、环境、数据哈希和独立报告复现？

本知识库第一版不注册远端 agent，不读取 Cookie/Token，不连接 broker，不同步/复制交易，不自动下单。未来即使扩展，也必须先在无资金沙盒中把“研究建议”和“订单执行”做硬权限隔离，并保留人工确认。

## 本地扫描结果怎么读

`investing-research` 插件扫描非敏感文本的相对路径，只输出类别计数，不输出源码行。固定 commit 的扫描规模：OpenBB 1,925、myhhub/stock 108、AI-Trader 137、ai_quant_trade 754 个文本文件。OpenBB 的 provider 信号 1,498 个路径反映模块化规模；ai_quant_trade 的 backtest 信号 98 个路径反映材料广度；这些数字都不是测试覆盖率。

扫描有 false positive 和 false negative。文件名或 README 提到 backtest 可能只是文档；没有出现 `lookahead` 字样也不表示没有未来函数。正确流程是：计数发现线索 → 打开最小关键文件 → 找测试和 fixture → 独立手算 → 写采用/拒绝决策。

## 插件使用与采用边界

插件由 `.agents/plugins/marketplace.json` 发布，Skill 与扫描器位于 `plugins/investing-research/`。它只接受已经合法克隆的本地目录，跳过 `.env` 及 Cookie、Token、credential、secret 文件，不发网络请求、不导入上游模块、不修改 clone。

采用优先级：

1. **直接采用思想**：OpenBB 标准模型/provider 分层，项目级来源谱系，策略后验收。
2. **独立重写并测试**：指标公式、债券/组合/回测小算法、数据 schema。
3. **可选依赖**：许可证清晰、用户自行安装、默认只读、离线 CI 有替身。
4. **link-only**：许可证未知、数据权不清、远端 Skill 或营销材料。
5. **拒绝默认启用**：Cookie 绕过、broker/复制交易、自动下单、杠杆/高频和无人工确认。

## 审计检查清单

1. URL、pinned commit、日期、根许可证和嵌套许可是否齐全？
2. 代码事实、上游声明和我们的推断是否分栏？
3. 数据源授权是否与代码许可分开？
4. 复权、时区、公司行动、历史时点、缓存与 schema 是否有测试？
5. 回测的费用、滑点、次日成交、退市、样本外和尝试数是否披露？
6. 安装是否执行脚本、容器、远端 Skill 或读取认证？
7. 自动交易面是否彻底隔离，默认是否不下单？
8. 有哪些明确未知，下一步最小安全测试是什么？

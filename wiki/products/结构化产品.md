# 结构化产品

结构化票据通常是发行人的无担保债务加嵌入衍生品。营销名称中的“本金保护”“收益增强”必须翻译成逐情景现金流，且任何保护都受发行人信用约束。

## 条款拆解

1. 债务：发行人/担保人、顺位、到期、信用、提前赎回与二级流动性。
2. 参考资产：指数/股票/利率/商品/汇率、价格来源、公司行动和中断事件。
3. 路径：观察日、连续/收盘障碍、敲入/敲出、最差标的、票息记忆和自动赎回。
4. 收益：参与率、上限、下行承担、币种、费用与税。

```python
from investkb.assets import structured_note_redemption

result = structured_note_redemption(
    [100, 65, 80], principal=1_000, barrier_ratio=0.70,
    participation=1.0, cap=0.15, issuer_recovery=0.40,
)
```

该练习只表示一个“路径触碰障碍且期末下跌”的教学条款，再把合同支付乘以发行人回收率；不是任何真实产品估值。至少同时运行：温和上涨、超过上限、盘中/观察日触障、期末恢复、期末下跌、发行人违约、提前赎回和无法卖出。

## 常见失败

- 只看票息/上限，不画全路径支付函数。
- 把发行人估值或“本金保护”当第三方担保。
- 忽略发行价高于估计公允价值、销售/对冲成本和二级市场折价。
- 用标的历史回测却漏掉观察日、公司行动、提前赎回和幸存产品偏差。

如果不能独立复述每个情景的支付、信用和流动性后果，就还没有理解产品。参见 [[衍生品]]、[[期权与波动率]]、[[债券]]、[[流动性风险]]。

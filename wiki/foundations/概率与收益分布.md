# 概率与收益分布

## 学习目标

能够把情景写成互斥且穷尽的结果与概率，计算离散随机变量的期望和方差，理解期望收益不是最常发生的收益，并识别正态假设在金融尾部风险中的局限。

## 核心概念

随机变量把不确定结果映射成数值。离散情景 (x_i) 的概率为 (p_i)，必须满足 (p_i\ge0) 且总和为 1。期望 (E[X]=\sum p_i x_i) 是长期概率加权中心，不是下一期保证；方差 (Var(X)=\sum p_i(x_i-E[X])^2) 衡量围绕期望的离散程度，但不会告诉你损失发生在哪一侧。

相关性描述线性共同变化，不等于因果，也不在危机中保持不变。收益分布常有偏度、厚尾、波动聚集和制度断点。均值与标准差仍是有用的第一层摘要，但不能把“均值减两倍标准差”机械解释成极端损失上限。

条件概率强调信息集：(P(A|B)=P(A\cap B)/P(B))。研究财报后上涨概率时，B 必须是当时能观察到的规则，不能事后按涨跌选择“好财报”。贝叶斯公式可以更新信念，但先验、似然和数据生成过程都需披露。

## 手算例

假设一个教学资产只有两种一年结果：40% 概率亏损 10%，60% 概率盈利 20%。期望收益为 (0.4\times(-0.10)+0.6\times0.20=8\%\)。方差为 (0.4(-0.18)^2+0.6(0.12)^2=0.0216)，标准差约 14.70%。下一年仍可能亏 10%；“期望 8%”不是账户自动增长 8%。

如果连续两年独立重复，两个“都盈利”的概率是 (0.6^2=36\%\)。真实市场年度结果通常不独立：经济周期、估值和政策会让条件概率变化，所以独立假设必须写出来，而不是默认。

## Python 实验

```python
from investkb.education import discrete_moments

mean, variance = discrete_moments([-0.10, 0.20], [0.40, 0.60])
assert round(mean, 4) == 0.0800
assert round(variance, 4) == 0.0216
```

把概率改成 `[0.4, 0.4]` 会抛错，因为缺失的 20% 情景会让期望失去定义。显式失败能防止电子表格悄悄接受不完整情景。

## 投资场景

估值情景、信用违约、药物审批、商品供给和公司诉讼都可先用情景树表达。关键不是制造漂亮的概率，而是列出结果是否穷尽、概率依据、共同驱动和更新条件。对于组合，还要考虑资产之间的联合分布；把每只股票单独“胜率”相乘通常没有意义。

## 常见误用

- 把期望收益当最可能收益或保证收益。
- 用正态分布消除厚尾、停牌、涨跌停和流动性断裂。
- 把零相关当独立，把相关当因果。
- 用事后数据定义条件，再声称条件提高胜率。
- 给情景概率很多小数，却不披露依据和敏感性。

## 检查清单

1. 情景是否互斥、穷尽，概率是否和为 1？
2. 概率来自历史频率、市场价格、模型还是主观判断？
3. 历史样本是否包含危机、退市和制度变化？
4. 均值之外是否报告损失侧、偏度、尾部和最大可能损失？
5. 更新信息在决策时是否可得？

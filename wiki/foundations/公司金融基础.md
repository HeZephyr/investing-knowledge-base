# 公司金融基础

## 学习目标

理解公司如何在投资、融资和分配之间配置资本，能用净现值而非会计利润判断简单项目，区分企业价值与股权价值，并识别资本成本、稀释、杠杆和管理层激励对每股价值的影响。

## 核心概念

公司金融的核心不是“融资技巧”，而是把有限资本投向风险调整后净现值为正的用途。项目净现值 (NPV=\sum CF_t/(1+r)^t-I_0)。内部收益率是使 NPV 为零的贴现率，但非常规现金流可能有多个 IRR，不同规模项目也会让 IRR 排名误导；NPV 通常更直接连接价值增量。

企业价值属于债权人与股东等全部资本提供者，股权价值是扣除净债务和其他优先索取后的剩余。经营利润、自由现金流和每股价值必须使用一致口径。公司用债务融资可能获得税盾，却增加固定承诺、再融资和困境成本；增发降低杠杆，也可能稀释现有股东。

资本配置包含维持性/增长性资本开支、研发、并购、还债、分红和回购。回购只有在价格相对内在价值、替代用途和资产负债表约束合适时才创造每股价值；用债务高价回购可能反向毁损。管理层的增长目标、奖金和任期会影响决策，治理不能从模型中省略。

## 手算例

项目今天投入 100，一年后确定回收 112；若同风险资本成本为 10%，现值 112/1.10=101.82，NPV=1.82。若回收只是预测，应列低/中/高现金流和概率，不能把 112 当确定。

另一个每股例：公司价值 1,000，净债务 200，股权价值 800；有 100 股时每股 8。若新增 20 股只换来 100 现金且现金没有创造额外净现值，股权价值成为 900、股数 120，每股 7.5，原股东被稀释。真实交易还要考虑发行价格、用途和控制权。

## Python 实验

```python
from investkb.education import present_value

project_value = present_value(112, annual_rate=0.10, periods=1)
npv = project_value - 100
assert round(npv, 2) == 1.82
```

这是单一现金流教学。多期、税、营运资本、终值和风险情景应显式建模，不要用一个函数隐藏口径。

## 投资场景

阅读年报时，把现金流量表与管理层资本配置逐项连接：经营现金是否足以维持业务，资本开支产生何种增量回报，并购商誉和整合目标是否兑现，回购是否抵消股权激励，债务何时到期。对银行和保险，普通工业公司的自由现金流定义不适用，应回到资本与监管框架。

## 常见误用

- 用收入增长或 EPS 增长自动等同价值创造。
- 用历史平均回报直接当项目资本成本。
- 企业价值、股权价值、净债务和少数股东口径混用。
- 把全部资本开支视为增长，忽略维持性投入。
- 认为任何回购都增厚，忽略价格、债务和股权激励。
- 只用管理层给出的“调整后”指标，不与法定报表勾稽。

## 检查清单

1. 项目现金流是增量、税后且包含营运资本和终值吗？
2. 贴现率与币种、期限和风险匹配吗？
3. 企业价值如何桥接到股权价值和完全摊薄每股价值？
4. 资本开支、研发、并购、分红、回购和还债的替代成本是什么？
5. 管理层激励、关联交易和融资到期会怎样改变情景？

# 论文阅读与复现

## 学习目标

能把论文拆成问题、理论机制、数据、识别/预测方法、结果和外部效度，区分“复算代码成功”与“结论可在另一时期成立”，并用预注册、环境固定、数据哈希和负结果记录建立最小复现包。

## 核心概念

先读摘要和结论只能知道作者主张，不能知道证据强度。第一遍应写一句可证伪命题；第二遍画数据谱系：总体、样本、时间、字段、当时可得日、清洗和排除；第三遍检查方法是否回答问题；最后才读表格中的经济量级、区间和稳健性。

复现分层：计算复现是在同数据同代码下得到同结果；方法复现是独立实现得到近似结果；直接复现是在新样本重复同设计；概念复现用不同操作化检验同机制。运行作者 Notebook 只接近第一层，而且依赖、随机种子、外部下载和数据许可都可能阻断。

金融研究特别容易受历史时点、幸存者偏差、复权、退市收益、多重尝试和费用影响。论文样本期后的“扩展”是新检验，不应静默覆盖原始结果。负结果必须保留，否则团队会重复搜索同一个无效空间。

## 手算例

某论文报告毛收益每月 0.40%，策略单边换手 60%，每边综合成本 0.30%。若粗略按往返换手成本 (2\times0.60\times0.30\%=0.36\%\)，费用后只剩 0.04%，还未计冲击、税和容量。这个简算不能替代逐笔模型，但足以阻止直接把毛收益当可实现收益。

若作者测试 100 个信号却只报告 5 个，单个 p 值必须放回完整尝试集合。复现记录要保留所有 100 个结果或至少明确无法知道尝试数。

## Python 实验

```python
from investkb.education import bonferroni_threshold, rolling_origin_splits

alpha = bonferroni_threshold(0.05, 100)
splits = rolling_origin_splits(120, min_train_size=60, test_size=12, step=12)
assert alpha == 0.0005
assert all(max(train) < min(test) for train, test in splits)
```

代码只建立多重比较和时间切分底线。完整复现还需锁定数据、环境、种子、命令和预期产物；不能复制 ISLP Notebook 充当自己的验证。

本库已有一个公开冻结数据复现：`raw/cases/factor-strategy/manifest.yaml` 保存 Kenneth French Data Library 的来源 URL、2026-05 数据 vintage、行数、单位、决策日与文件 SHA-256；`evidence_metrics(..., analysis="factor")` 独立计算 2000—2025 年 HML 算术均值、复利累计收益、正收益年数，并由 `tests/test_public_evidence_cases.py` 锁定预期值。结果展示在 [HML 因子复现](../../output/cases/factor-replication.md)。

这叫**限定窗口的方法复算**，不是完整重现 CRSP/Compustat 个股组合构造，也不是证明价值因子未来有效。若把窗口延长、换市场、改权重或加入费用，必须另标为“扩展”，保留原复现产物，重新预注册命题与失败条件。

随机方法还要显式固定局部生成器：

```python
from investkb.advanced_foundations import seeded_bootstrap_mean

first = seeded_bootstrap_mean([1, 2, 4, 8], resamples=1000, seed=20260718)
second = seeded_bootstrap_mean([1, 2, 4, 8], resamples=1000, seed=20260718)
assert (first == second).all()
```

相同种子只是复现抽样路径；解释环境还需 `pyproject.toml`/锁文件、Python 版本、命令、操作系统相关性和外部数据哈希。不能用全局随机状态让测试顺序改变结果。

## 投资场景

复现价值、动量、质量、低波或宏观择时论文时，先问数据是否免费合法取得，是否包含退市证券，财务字段何时公布，组合何时调仓，基准和无风险利率如何定义。若关键专有数据不可得，可以做方法演示，但必须把结论降级为“无法独立验证”，不要用近似数据假装同一研究。

## 常见误用

- 只运行作者代码一次就声称结论已复现。
- 用今天修订后的历史数据替代论文当时数据。
- 调整参数直到结果接近论文，却不记录尝试。
- 只复现毛收益，不复现基准、费用、退市和容量。
- 仓库无许可证仍复制 Notebook 或数据。
- 失败后删除分支、结果和原因，制造幸存者偏差。

## 检查清单

1. 可证伪命题、主要指标和失败阈值是什么？
2. 数据许可、版本、截止日、可得日、哈希和排除规则齐全吗？
3. 环境、随机种子、命令、运行时间和预期文件是否固定？
4. 尝试数、负结果、费用、基准和样本外是否报告？
5. 当前完成的是计算、方法、直接还是概念复现？不能外推到哪里？

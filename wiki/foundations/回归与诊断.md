# 回归与诊断

## 学习目标

理解一元 OLS 的斜率、截距和残差，能用手算直线核对实现，区分描述、预测与因果，并在读取系数和 (R^2) 前检查函数形式、异常点、异方差、自相关和样本外稳定性。

## 核心概念

模型 (y_i=\beta_0+\beta_1x_i+\epsilon_i) 用最小残差平方选择系数。一元斜率等于 x 与 y 的共同变化除以 x 的平方变化，截距让拟合线经过样本均值。残差是观测与拟合值之差；它不是不可观察的真实误差，但能暴露模型遗漏。

(R^2) 是样本内方差解释比例，不是因果概率、预测准确率或策略收益。高 (R^2) 可能来自共同趋势、泄漏或机械恒等式；低 (R^2) 也不表示一个小但稳定的条件均值没有决策价值。应报告单位、系数区间、残差图、基准误差和时间后验证。

因果解释还要求识别假设：没有遗漏共同原因、反向因果、选择机制或错误测量等。增加控制变量不自动解决问题，错误控制碰撞变量或处理后变量反而会制造偏差。金融面板还可能存在公司内相关、日期共同冲击和横截面依赖，需要合适的聚类或稳健标准误。

## 手算例

x 为 1、2、3，y 为 3、5、7。x 均值 2，y 均值 5；共同离差和为 4，x 离差平方和为 2，因此斜率 2，截距 1，残差全为 0，(R^2=1)。这个完美例子只证明算术：若 x 是年份、y 是随通胀上升的价格，完美直线仍不证明年份导致公司价值。

加入一个极端点 (4, 50) 会大幅改变斜率。研究者应解释它是数据错误、真实尾部还是制度切换；不能只因为它破坏显著性就删除。

## Python 实验

```python
from investkb.education import simple_ols

fit = simple_ols([1, 2, 3], [3, 5, 7])
assert fit.intercept == 1.0
assert fit.slope == 2.0
assert fit.r_squared == 1.0
```

教学函数拒绝无 x 变化和无 y 变化的样本，避免输出不可解释的斜率或 (R^2)。真实研究可用 statsmodels 获得稳健协方差与诊断，但必须明确版本和选项。

## 投资场景

常见应用包括估计股票对市场收益的 beta、商品价格对公司利润的敏感度、估值倍数与增长的横截面关系。每个场景都要对齐日期和信息集：季度利润不能用季度结束后很久才公布的数据提前回归交易信号。beta 也会随窗口、频率、市场状态和汇率变化，不是公司永久属性。

## 常见误用

- 把相关或显著系数写成因果结论。
- 用价格水平对价格水平回归而不检查趋势和伪回归。
- 只看 (R^2) 和 p 值，不画残差、不报告单位。
- 在全样本标准化或选特征后再切训练测试，产生泄漏。
- 任意删除异常点，或忽略同公司和同日期的相关误差。
- 用修订后财务数据回填公告前日期。

## 检查清单

1. 问题是描述、预测还是因果？识别假设分别是什么？
2. x、y 的单位、时间戳和可得日对齐吗？
3. 残差是否显示非线性、异方差、自相关或异常点？
4. 标准误是否匹配数据依赖结构？
5. 系数在时间后样本、其他市场和合理窗口下稳定吗？

# 假设检验与多重比较

## 学习目标

能写出零假设、备择、统计量和拒绝规则，正确解释 p 值，区分统计显著、效应量和经济价值，并在尝试多个指标、参数或股票时控制多重比较。

## 核心概念

p 值是在零假设及其模型假设成立时，观察到当前或更极端统计量的概率。它不是“零假设为真的概率”，也不是策略未来赚钱概率。显著性水平 \(\alpha\) 是分析前的错误容忍规则；看到结果后再移动阈值会破坏其含义。

第一类错误是假阳性，第二类错误是假阴性。检验功效取决于样本量、噪声、真实效应和阈值。只追求 p<0.05 会忽略效应是否足以覆盖手续费、滑点、税和模型衰减。应同时报告效应量、区间、基准和样本外结果。

当同时测试 (m) 个命题，每个都用 5% 阈值，至少一个偶然显著的机会会快速增加。Bonferroni 用 \(\alpha/m\) 控制族错误率，保守但透明；Benjamini-Hochberg 控制错误发现率，适合发现型研究，但仍需完整披露尝试集合。不能只对“入选结果”计算校正。

## 手算例

若研究者测试 20 个独立因子并希望整个实验的族错误率不超过 5%，Bonferroni 单项阈值为 0.05/20=0.0025。一个 p=0.01 的因子按单次检验看似显著，但在这组 20 次尝试中不能通过该规则。

经济量级也必须手算：若估计月度超额收益 0.15%，往返费用与滑点合计 0.20%，即便统计上显著，简单净效应仍为负。显著性不能支付交易成本。

## Python 实验

```python
from investkb.education import bonferroni_threshold

threshold = bonferroni_threshold(0.05, 20)
assert threshold == 0.0025
```

代码要求显式传入测试次数，目的是让“我到底试了多少次”成为数据。SciPy 可以计算具体统计检验，但研究问题、依赖结构、效应量和校正规则不能由函数自动决定。

## 投资场景

选股平台一次展示 200 个条件，回测者再尝试多个持有期、市场、费用、起止日和止损参数，真实尝试数远超最后报告的一条策略。预注册应在看到结果前固定主要命题、指标、方向、样本切分和失败阈值；探索结果可以保留，但必须标注为探索并在新样本验证。

## 常见误用

- 说“p=0.03，所以策略有 97% 概率有效”。
- 只报告显著结果，不披露失败的指标和参数。
- 把统计显著当成费用后经济显著。
- 数据窥探后再声称检验是预先设定。
- 多重校正只计算展示出来的候选，不含被删除的尝试。
- 未拒绝零假设就宣称“证明没有效应”。

## 检查清单

1. 零假设、备择、方向和统计量是否在看结果前写定？
2. 一共尝试了多少特征、窗口、市场和参数？
3. 报告效应量、区间和费用后结果了吗？
4. 样本依赖、异方差和分布假设是否处理？
5. 是否有真正未参与选择的时间后样本？

# 金融与统计基础

这不是一份“看完就会投资”的书单，而是一条从现金流到证据判断的训练路径。每个模块都要求四件事：能解释、能手算、能用小代码复核、能指出何时不能用。学习结果不以记住公式衡量，而以能否拒绝错误输入、写出假设和保留不确定性衡量。

## 推荐顺序

1. [[复利与贴现]]：先统一时间、频率和现金流时点。
2. [[概率与收益分布]]：理解情景、期望、方差和厚尾。
3. [[抽样与估计]]：区分总体、样本、估计量和误差。
4. [[假设检验与多重比较]]：避免把偶然显著当成策略。
5. [[回归与诊断]]：先检查残差、影响点与识别，再读系数。
6. [[投资数学与优化]]：用矩阵、导数、凸性与约束表达问题，并测试输入稳定性。
7. [[微观经济与产业组织]]：把供需、弹性、成本与竞争结构转成公司证据。
8. [[宏观账户周期与货币传导]]：勾稽账户、发布日期、政策工具和资产传导。
9. [[时间序列与预测]]：只用过去预测未来，并与简单基线比较。
10. [[债券与利率基础]]：把贴现落到固定现金流、久期和再投资风险。
11. [[公司金融基础]]：连接投资、融资、资本成本和每股价值。
12. [[财务三表]] 与 [[盈利质量]]：把会计恒等式、应计判断和现金转换连起来。
13. [[金融危机与机制复盘]]：用资产负债表和市场 plumbing 取代事后故事。
14. [[论文阅读与复现]]：把问题、数据、方法、结果与外部效度拆开。

## 三轮学习法

- 第一轮只做每页的手算例和检查清单，不碰真实证券。
- 第二轮运行 Python 片段，主动改成非法输入，观察为什么必须失败。
- 第三轮选一张公开冻结数据表，先预注册问题和基准，再做小型复现；失败结果同样保留。

## 表格与随机计算底线

表格计算先声明分组键、数值列、缺失值和单位，不依赖“当前行顺序正好如此”。教学函数会保留分组首次出现顺序，并拒绝缺列、无穷值和不可哈希分组：

```python
from investkb.advanced_foundations import grouped_table_summary, seeded_bootstrap_mean

rows = [
    {"market": "CN", "return": 0.1},
    {"market": "HK", "return": -0.1},
    {"market": "CN", "return": 0.3},
]
summary = grouped_table_summary(rows, group_key="market", value_key="return")
bootstrap = seeded_bootstrap_mean([0.1, -0.1, 0.3], resamples=1000, seed=20260718)
assert summary[0]["count"] == 2
assert len(bootstrap) == 1000
```

真实项目还必须固定数据 schema、Python 与依赖版本；显式种子只固定随机路径，不固定上游数据和数值库变化。

## 完成标准

内容就绪不代表会做题，会做题不代表做过真实案例，做过一次案例也不代表关系会持续。机器覆盖审计分别记录 `content-ready`、`exercise-tested`、`case-validated` 和 `maintenance-live`。本课程先完成前两类；真实投资命题仍必须通过历史可得数据、费用、样本外和负结果验证。

## 安全边界

所有例子都是教学用的合成数字，不构成投资建议。利率、收益率、波动率和回归系数在不同币种、频率、复权口径和市场制度下不可直接比较。看到“精确答案”时，先问现金流时点、数据截止日、样本选择和替代解释。

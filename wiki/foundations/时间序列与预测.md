# 时间序列与预测

## 学习目标

能区分解释与预测，识别趋势、季节、自相关、结构变化和修订，建立朴素基线，并用严格按时间前后排列的滚动起点切分评价预测，不让未来观察进入训练。

## 核心概念

时间序列顺序携带信息，不能像普通表格一样随机打乱。预测任务必须写清变量、预测起点、期限、发布延迟、使用者和损失函数。一步预测与一年预测面对不同可预测成分；库存预测、利润预测和股票收益预测也不能共享同一成功标准。

基线是最低诚信要求。水平序列可用最后值，季节序列可用上年同期，收益序列常用零或历史均值。复杂模型只有在未见未来数据、相同信息集和相同损失函数下稳定优于基线，才值得讨论。滚动起点评价在每个预测起点只用此前数据，扩大或移动训练窗，再评价后续窗口。

平稳性不是“图看起来平”。均值、方差和依赖结构可能随通胀、制度、竞争和危机变化。差分可去除某些趋势，却也改变问题；协整可能描述长期关系，但不能免除因果与结构断点检验。预测区间必须反映残差与参数不确定性，金融尾部还可能超过模型分布。

## 手算例

有 9 个按月排列的观测，最小训练 4 个月、每次预测 2 个月、步长 2。第一轮用索引 0–3 预测 4–5；第二轮用 0–5 预测 6–7；索引 8 不足两个完整测试月，因此不计入。任何一轮训练索引最大值都必须小于测试索引最小值。

若朴素模型的 MAE 是 5，复杂模型是 4.9，但后者在不同起点有时恶化到 12，还需要考虑稳定性、维护成本与决策损失，不能只凭平均改善 0.1 宣称胜利。

## Python 实验

```python
from investkb.education import rolling_origin_splits

splits = rolling_origin_splits(9, min_train_size=4, test_size=2, step=2)
assert splits[0] == ((0, 1, 2, 3), (4, 5))
assert all(max(train) < min(test) for train, test in splits)
```

函数只负责切分纪律，不替你选择模型或损失。statsmodels 是实现参考；FPP3 主要使用 R，概念可迁移但 API 不可照抄。

## 投资场景

能源库存、零售额、公司订单和利润率通常比短期股票收益更有可解释结构，但会修订且有公布延迟。研究者应保存 release vintage，并把“预测经济量”与“经济量公布后能否交易”分开。对股票收益，零预测可能是很强基线；即使方向准确，也必须加入换手、滑点、涨跌停和容量。

## 常见误用

- 随机切分时间序列，让未来出现在训练集。
- 全样本去趋势、标准化或选参数后再回测。
- 不与最后值、季节朴素或零收益基线比较。
- 用今天下载的最终修订序列模拟过去预测。
- 只报告一个起点和一个误差指标。
- 把预测区间当真实结果必然落入的保证区间。

## 检查清单

1. 预测变量、起点、期限、发布时间和损失函数是什么？
2. 每个训练样本在当时都可得吗？预处理是否只在训练内拟合？
3. 基线是什么，复杂模型在多少滚动窗口稳定胜出？
4. 是否保存 vintage、修订、缺失和异常处理？
5. 预测改善能否覆盖交易或业务执行成本？

# 周期行业研究

周期行业的收入、利润、库存和资本开支随供需缺口摆动。低市盈率可能来自周期顶部的异常高利润，高市盈率也可能出现在利润底部，因此不能只按静态倍数排序。

## 通用框架

1. 需求：终端数量、单机含量、替代和库存变化。
2. 供给：产能、良率、投产周期、退出成本与行业集中度。
3. 价格：现货、合约价、产品组合和议价权。
4. 成本：固定/变动成本、原料、能源、汇率与折旧。
5. 资本纪律：资本开支、自由现金流、负债和股东回报。

先给每条业务记录分类体系、修订版、活动代码、实体映射生效日和主要活动。这里的日期是研究者或本地统计机构从何日开始使用该映射，不是 UNSD 为全球公司统一设定的“ISIC 生效日”。UNSD 明确 ISIC 是经济活动分类，不是机构部门或证券分类；多元公司要从法定分部重建，不能给整个集团贴一个标签后停止研究。

```python
from datetime import date
from investkb.sectors import IndustryClassification

record = IndustryClassification(
    "ISIC",
    "Rev.5",
    "2610",
    "电子元件与电路板制造",
    mapping_effective_from=date(2026, 7, 18),
)
```

## 先判断行业类型

| 行业类型 | 第一问题 | 典型应用 |
|---|---|---|
| 商品/产能周期 | 库存、成本曲线、供给滞后如何定价？ | [[能源]]、[[存储半导体]] |
| 资产负债表/信用周期 | 资金、期限、损失与资本如何联动？ | [[金融]] |
| 品牌/渠道与单位经济 | 量价结构、渠道库存和再投资回报如何变化？ | [[消费]] |
| 长周期订单/安装基数 | 订单如何转成积压、收入、产能和服务？ | [[工业与资本品]]、[[硬件与半导体产业链]] |
| 项目/自然资源 | 土地、建设、品位、回收和融资如何转成现金？ | [[地产开发与运营]]、[[材料与矿业]] |

通用框架只提供问题，不提供答案。每个应用都必须建立“指标 → 财务 → 估值”映射，并写一个宏观判断正确但公司结论错误的反例。

## 验证门槛

页面或空白模板不等于框架已经验证。至少需要三个使用冻结公开数据的完整案例，记录历史可得日、数据修订、基准、失败条件和正负结果，才能把框架状态提升为 validated。失败案例必须保留在 [[经验与失败教训]]。

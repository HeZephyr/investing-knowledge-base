# 投资政策与目标

投资政策书先回答钱何时要用、什么损失会破坏生活、哪些产品有能力理解，再讨论收益。公开模板只展示字段，不记录个人账户、持仓、身份或密码。

## 最小合同

- 目标：保值、现金流、长期增长或学习实验，不能只写“赚钱”；
- 期限：每个目标的日期与中途现金需求；
- 安全垫：紧急备用金和不可投资负债；
- 能力/意愿：财务上能承受的损失与心理上愿意承担的波动分别写；
- 边界：允许资产、禁止产品、杠杆、单资产和权益上限；
- 治理：决策日、下一复核日、触发临时复核的生活事件和谁有权限修改。

`validate_ips` 对公开示例校验必填字段、权重范围和复核日期。它不能替用户决定合适比例；用户当前无持仓，私人 IPS 继续留在 Git 忽略层。

## 年度复核

先检查目标、现金需求、税务/法律环境和承受能力是否变化，再检查组合偏离。市场涨跌本身不是修改长期政策的充分理由。所有政策变更写修改原因、旧规则、新规则和生效日，禁止事后覆盖。

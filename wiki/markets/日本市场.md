# 日本市场

日本市场研究要同时回答“在哪交易、何时交收、公司何时披露、外国投资者通过什么账户持有、日元风险如何进入回报”。JPX 是交易所集团，TSE 是股票交易场所，JSCC 是中央对手方，JASDEC 负责簿记转移，FSA/EDINET 承担监管与法定披露入口；这些角色不能混为一个网站。

本页是学习框架，不构成投资建议。英文材料可能晚于日文正式文本，任何真实交易前必须向合规券商和当期官方规则复核。

## 新手先知道

- 日经 225 与 TOPIX 是不同编制逻辑的指数，不代表“日本全部股票”；研究个股先确定市场板块、普通股/优先股、交易单位和指数权重来源。
- 日元计价上涨不等于人民币或港币计价获利，必须拆出本地股票收益、JPY/CNY 或 JPY/HKD 汇率与换汇成本。
- 日本公司交叉持股、母子上市、现金余额、治理改革和回购是研究变量，不是“日本价值股一定上涨”的结论。
- 海外投资者通常经券商和托管链访问；本仓库不推荐券商，不保存账户、税号或持仓。

## 制度或现金流地图

订单从券商进入 TSE 的 arrowhead/场外大宗机制，成交后由 JSCC 承接和净额清算，再经 JASDEC 与现金系统交收。2026-07-17 官方入口显示普通交易按 T+2 处理；上午、下午、收盘机制、特别报价、涨跌幅和报价单位均以对应日期规则为准。

公司研究从 EDINET 的年度/半年度证券报告、临时报告和大量持有报告开始，再与 JPX/TSE 适时披露交叉。法定报告期、提交时间和更正关系比网页发布日期更重要。分红、拆股、配股、要约收购和退市必须单独建立公司行动事件，不能仅靠复权价格反推。

税费至少拆为券商佣金/平台费、交易场所费用、换汇价差、股息预扣、资本利得与投资者居住地税务。税率会随身份、协定和账户变化，本页不写固定“万能税率”。

市场板块和公司类型也影响可比性：Prime/Standard/Growth 的上市要求与流动性不同，REIT、ETF 和外国股票适用产品规则；“一手 100 股”等常见描述必须按证券和日期核验。任何板块迁移、代码变更、股份合并或拆分都进入样本谱系。

## 数据与历史时点

最小行情记录：证券代码及代码变更、venue、交易日、日本时区、OHLCV、成交额、币种、复权状态、公司行动来源、抓取时间和哈希。最小披露记录：EDINET document ID、提交人法律实体、报告期、提交时间、更正链、taxonomy 版本和原始语言。

免费网页或 API 没有投资研究 SLA。EDINET API v2 需要注册 Key；Key 只存在本地环境，不能用 Cookie 绕过、不能写入 URL、日志或 Git。今天下载的更正后 XBRL 不能冒充历史交易日已知数据。

## 可执行练习

1. 选一家无持仓关系的公司，记录某年度报告首次提交时间与一次更正（若有），只使用首次提交前后分别可得的字段建立两个快照。
2. 用同一段股票本地收益，分别叠加 JPY 升值、贬值和不变三种汇率路径，比较投资者币种回报。
3. 为“收盘信号次日成交”写规则卡：明确日本时区、下午收盘、T+2 交收与假日，不把交收日误当成交日。

练习只验证数据与制度理解，不产生买卖信号。

## 常见失败模式

- 用当前 TOPIX 成分回测二十年前市场，造成幸存者偏差。
- 把英文摘要当完整法定披露，遗漏日文附件、修订或风险项。
- 把日元弱势简单等同出口企业利润上升，忽略海外成本、套保、地区收入和价格传导。
- 把 TSE 普通股规则用于 ETF、REIT、外国股票、信用交易或衍生品。
- 将当前交易时段、收盘竞价和报价单位回填到规则变更前历史。

## 研究检查清单

1. 法律实体、证券类别、市场板块、交易单位是否一致？
2. 使用的是日文正式规则还是仅参考英文译本，版本和生效日是什么？
3. EDINET 披露的首次可得时间、更正链和 XBRL taxonomy 是否保留？
4. 汇率、预扣税、托管、佣金和交收日是否与投资者身份匹配？
5. 公司行动、停牌、涨跌幅、不可成交与退市是否进入回测？

## 案例骨架

命题示例：“治理改善后资本配置提高”必须预注册指标（现金/市值、回购注销、ROIC、交叉持股变化）、比较组和三年后失败条件。证据按“政策/交易所变化 → 公司董事会决策 → EDINET 当时披露 → 三表现金流 → 估值”连接。至少保留一个治理指标改善但股东回报未改善的反例，避免把国家叙事替代公司证据。

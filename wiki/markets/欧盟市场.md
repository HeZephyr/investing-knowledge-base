# 欧盟市场

欧盟提供统一法规层，但不是一座交易所、一个证券账户或一个税区。研究必须从 EU 规则继续下钻到发行人的 home member state、国家主管机关、实际交易 venue、CCP/CSD、币种和投资者税务。英国已经不是欧盟成员，本页不能替代英国 FCA/LSE 规则。

本页用于学习与研究，不构成投资建议、法律或税务意见。

## 新手先知道

- MiFID II/MiFIR 定义受监管市场、MTF、OTF 和系统内部化商；同一证券可在多个场所交易，主上市地不一定等于成交发生地。
- Transparency Directive、Prospectus Regulation、ESEF 与成员国机制共同决定披露。ESAP 正在分阶段建设，不能假设历史材料已由一个 API 完整统一。
- 欧元区、欧盟和“欧洲”范围不同；股票可能用 EUR、DKK、SEK、PLN 等币种计价。
- UCITS 是产品监管框架，不代表基金低风险、低费、无税务或适合任何投资者。

## 制度或现金流地图

法规层由欧盟法规/指令、委员会 delegated acts 和 ESMA RTS/ITS/指南组成；成员国主管机关监管发行人与中介，交易场所执行订单规则，CCP/CSD 完成清算结算。对每笔研究对象建立：发行人注册地 → home member state → 披露门户 → ISIN → venue/MIC → CCP/CSD → 结算币种。

2026-07-17 的核验边界：现行一般结算仍以 T+2 理解，欧盟已政治同意向 T+1 转换，目标为 2027-10-11 且存在范围例外。计划日期不等于历史或当前已经生效；实现前后都要回到 Official Journal 和具体 CSD。

税费可能包括券商、venue/清算、印花/金融交易税、股息预扣、资本利得和换汇，且各国和投资者身份不同。本页拒绝一个“欧洲统一交易税”数字。

跨境持有还涉及发行人 CSD 与投资者 CSD、代理投票、公司行动选项和多语言通知。券商 UI 展示一行“欧洲股票”不能证明底层托管、税收减免或投票权路径相同。

## 数据与历史时点

行情数据保存 ISIN、MIC、交易币种、当地时区、交易状态、公司行动和 consolidated/venue-specific 标识。披露保存 ESEF 文件、发行人 LEI、报告期、公布时间、语言、taxonomy 与更正。跨 venue 合并时禁止把不同币种、交易条件和延迟报价直接相加。

ESMA 透明度参数按季度/年度计算，法规和技术标准也会修订。研究保存法规编号、生效/适用日、当期流动性分类和抓取时间；未来 ESAP 回补历史不表示旧交易日可得。

## 可执行练习

1. 任选一只无持仓关系的欧盟上市公司，画出法律实体、LEI、ISIN、home member state、披露门户和至少一个 MIC。
2. 模拟 T+2 与 T+1 下跨币种 ETF 申赎/证券成交的现金时间线，只比较流动性需求，不声称收益变化。
3. 为同一 ISIN 的两个 venue 记录币种、时区、交易量和价差，说明为何不能机械选“最便宜价格”。

## 常见失败模式

- 把欧盟、欧元区、欧洲、Euronext 和 STOXX 当同一范围。
- 把 T+1 政治协议提前写成当前结算规则，或忽略证券融资交易例外。
- 只抓英文聚合网站，遗漏成员国原始披露与发行人 home member state。
- 用一个 venue 的成交量代表全市场，或把延迟行情当实时可成交价。
- 因 UCITS 标签忽略跟踪误差、借券、衍生品、税务和基金注册地。

## 研究检查清单

1. EU 法规、成员国转置、主管机关与 venue 规则分别是什么版本？
2. ISIN/LEI/MIC、币种、时区和 CSD 是否完整？
3. 当前结算周期和未来转换是否按生效日分开？
4. ESEF 首次披露、更正、语言和 taxonomy 是否保存？
5. 税务与投资者身份是否由专业来源复核，而非网页估算？

## 案例骨架

命题示例：“多 venue 竞争降低交易成本”。预注册证券样本、venue、观察期、quoted/effective spread、成交规模、行情延迟和规则变化日；用未参与参数选择的时期检验。失败条件包括 consolidated 成本未下降、改善仅来自样本流动性上升或数据缺少场外成交。公司案例另从 ESEF 披露连接业务现金流，不能由市场结构直接推导盈利。
